% ========== 1. INTRODUCTION ==========
\section{Introduction}

\subsection{Purpose}
This document defines the test cases required to verify that the \textbf{mail deletion} feature of a web-based mail provider (modeled after Google Mail) functions correctly and meets expected behavior. The 10 test cases selected represent the highest-value coverage across core deletion flows, undo/restore, bulk operations, and negative scenarios.

\subsection{Scope}
The scope of testing covers:
\begin{itemize}[noitemsep]
    \item Deleting a single email from the Inbox
    \item Deleting multiple emails (bulk deletion)
    \item Undoing a deletion via toast notification
    \item Permanently deleting email from Trash
    \item Restoring a deleted email from Trash back to Inbox
    \item Deleting using the keyboard shortcut (\texttt{\#})
    \item Deleting all visible emails using Select All
    \item Deleting an email from search results
    \item Deleting an entire conversation thread
    \item UI validation: Delete button state when no email is selected
\end{itemize}

\subsection{Test Environment}
\begin{itemize}[noitemsep]
    \item \textbf{Browser:} Google Chrome (latest stable version)
    \item \textbf{OS:} Windows 10/11, macOS, Linux
    \item \textbf{Mail Provider:} Google Mail (Gmail)
    \item \textbf{Test Account:} Dedicated test Gmail account with $\geq$5 pre-loaded unread emails
    \item \textbf{Automation Stack:} Java 17+, Selenium WebDriver 4.x, TestNG 7.x, ChromeDriver (managed by WebDriverManager)
\end{itemize}

\subsection{Automation Coverage}
Of the 10 test cases, \textbf{2 are fully automated} (TC-DEL-001 and TC-DEL-002) using the Selenium/TestNG suite. The remaining 8 are documented for manual execution, as they require either long-running conditions, manual UI interaction, or environments not suitable for headless automation in this project scope.

\section{Risks and Assumptions}

\begin{itemize}[noitemsep]
    \item \textbf{Risk:} Gmail UI may change frequently; CSS selectors and aria-labels used in automated tests may break.
    \item \textbf{Risk:} Google may enforce CAPTCHA or additional verification (e.g., OTP), blocking automated login.
    \item \textbf{Assumption:} The test Gmail account has been pre-configured and contains at least 5 unread emails.
    \item \textbf{Assumption:} Tests are executed on the latest stable version of Google Chrome.
    \item \textbf{Assumption:} Network connectivity is stable during test execution.
    \item \textbf{Mitigation:} For Google verification challenges, use environment-configurable credentials (App Passwords or \texttt{.env} file) rather than hardcoded values.
\end{itemize}
